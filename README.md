## 神籤 - SHIN SEN -
___
## 1. サービス概要

このアプリは1日1回だけ神籤を引くことができるWebアプリです。

本アプリは単なる運勢表示ではなく、前向きなきっかけを与える装置として再定義したアプリです。

神籤結果には「行動タスク」が付与され、ユーザーは達成チェックを行い、履歴をカレン

ダー形式で振り返ることができます。

## 2. このアイデアはどこから生まれたか

>### 2-1. きっかけとなった体験・感情

日々、学習や仕事をしている中で、

- 毎日同じような仕事や作業の繰り返し

- 成果がすぐに見えない学習期間

- 気持ちが少し沈む日

など気持ちが停滞すると行動量が減り、自己肯定感も下がりやすくなります。

私の場合、ゲームのガチャを引く瞬間の「ワクワク感」で切り替えていましたが、その瞬間だけ

で終わってしまうのが勿体ないと感じていました。

またそれでも行動が停滞してしまう時は、ゲーム感覚で「お題」として行動指針を提示されると

ミッションに挑むような感覚で行動したりすることが多々あります。

こういった背景から「引く瞬間のワクワク」と「行動指針に対する達成感」を日常に組み込みた

いと考えました。

>### 2-2. なぜそれが「気になった」のか

ほんの少し前向きになることで行動は変わります。

しかし既存のおみくじは、引いて結果を見るだけという体験で止まっています。

そこで本アプリでは、未来を占うのではなく、「今日の行動を変える装置」

して神籤を再定義しました。

## 3. 課題の整理

>### 3-1. 表に見えている困りごと

- 日常が単調に感じる

- 気持ちが少し沈む日がある

- 行動するきっかけが少ない

>### 3-2. 本当に解決したい課題は何か

- 1回目のなぜ：
→ 単調だとモチベーションが下がるから

- 2回目のなぜ：
→ モチベーションが下がると行動量が減るから

- 3回目のなぜ：
→ 行動量が減ると自己肯定感がさらに下がるから

### 本当に解決したい課題

「気持ちの停滞防止」

## 4. 想定ユーザーについて

>### 4-1. 想定しているユーザー
■ 年齢設定

年齢：20〜30代

理由：仕事や学習に取り組みながらも、日々のモチベーション管理に悩む年代であるため。

■ 生活・状況設定

生活・状況：勉強中・就活中・仕事中など、日々努力を重ねている人

理由：成果が見えにくい期間に、小さな前向きなきっかけを与えられる。

■ 環境設定

環境：スマートフォンを日常的に使用している環境

理由：アプリはスマホ対応しており、隙間時間に気軽に利用できる。

>### 4-2. 自分とユーザーの距離

→ 過去の自分

学習や努力の途中で、少しだけ背中を押してくれるものが欲しかった経験があるためです。

>### 4-3. 実際に使われる可能性について

最初に使ってくれそうな人：友人や家族

代わりに使っているもの：SNSや動画アプリ

置き換えてまで使う理由：短時間で前向きなきっかけを得られるため

>### 4-4. 導入・継続の流れ

- 学習や仕事で、気持ちを切り替えたいと感じる

- ログインして1日1回神籤を引く

- 行動タスクを受け取る

- 達成したらチェック

- 履歴で積み重ねを確認

「1日1回」という制限があることで、無理なく習慣化できます。

## 5. 既存サービス・競合調査

>### 5-1. 似たサービス

- 結果表示のみ

- 行動管理なし

>### 5-2. それでも自分が作りたい理由

行動管理、振り返りを付与することで前向きになれる神籤アプリがないため

- 行動タスク付与

- 達成チェック

- 累積可視化

>### 5-3. 差別化を一文で

神籤を「未来予測」から「行動ログ」に進化させたアプリ。

## 6. このサービスで提供したい価値

>### 6-1. ユーザーの変化

行動の変化：気持ちを切り替えて行動を始めやすくなる

気持ちの変化：前向きになる

考え方の変化：積み重ねを可視化できる

>### 6-2. 価値を一文で表す

「日常に小さな前向きのきっかけを与え、成長を可視化するサービス」

## 7. 基本機能

>### MVPリリース

- ユーザー登録機能

- ログイン/ログアウト機能

- 1日1回抽選機能

- 行動タスク表示

- 達成チェック機能

- 履歴保存機能（日時、達成状態）

- レスポンシブ対応

>### 本リリース

- レア度別アニメーション演出強化

- 累積達成ボーナス解放（レア確率ブースト、特別演出）

- 達成回数による称号システム（カウントによる称号付与）

- SNS共有機能

- PWA対応

### 達成カウント設計（再設計ポイント）

当初は「連続ログイン型ストリーク」を検討しました。

しかし、1日未達でリセットは心理的負荷が高く、継続ハードルが上がるため累積型達成

システムに再設計しました。

- 神籤を引くだけでは達成にならない

- 行動タスク達成時のみカウント

- 未達は減少・リセットなし

- 累積方式

前進仕様（前向き）設計です。
